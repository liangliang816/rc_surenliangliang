
# 项目介绍
该项目基于gin框架开发
我认为这个是一个发布调阅模式的任务分发处理系统。
系统分为两个模块：
1、任务提交模块
用户提交任务时写入api_job表
2、任务处理模块
后台任务处理模块，负责从api_job表中读取未执行的任务，执行任务，将执行结果写入api_run_record表
主要实现代码在worker和controller目录中
系统实现需求的mvp版本，包括任务提交、任务处理、任务查询等功能。
但是还有以下可以迭代的优化点:
1、仅支持单线程处理任务，后续可以考虑支持多线程处理任务。
2、任务处理模块需要添加监控机制，检查主工作协程是否正常运行。
3、任务处理模块需要添加日志记录功能，记录任务处理过程中的关键事件和错误信息。
4、任务提交量大的时候可以加入消息队列，将任务提交到队列中，由后台任务处理模块异步处理。
再用Ai生成worker监控逻辑时，他只做了简单的监控处理，细节上没有实现，所以我没有使用Ai生成的监控逻辑。
